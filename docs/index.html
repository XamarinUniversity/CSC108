Before you begin

This is a Xamarin cross-platform class about C#. To complete the coding exercises, you will need a Xamarin development environment (Mac or Windows) to run the exercise on. Please read the setup requirements if you need help installing the Xamarin development environment.

Overview

In this course, you will get experience with the debugging process. You will set breakpoints so you can stop in the middle of execution to examine values and look at the call stack. You will use Watch Statements to track the values of your variables during a run of your app. Finally, you will learn to distinguish between exceptions and errors and use try/catch statements around potential exceptions.

Objectives

  1. Explain and illustrate the importance of debugging
  2. Discover how to throw and catch exceptions

Lecture

Learning C# with Xamarin: Debugging and Exceptions

Exercises

Exercise materials are available from GitHub. The code can be cloned through a desktop Git client or downloaded directly as a ZIP file.

  1. Set breakpoints and inspect values
    Exercise
    In this exercise, you will set breakpoints to tell the debugger to stop program execution at a set line. You will use the locals, watch, and call-stack windows to examine the state of your app while it is paused.
  2. Throw, catch, and handle an exception
    Exercise
    Here you will use an exception to indicate that a problem occurred during execution. You will throw an exception at the point you detect the fault and catch it in a different part of your code.
